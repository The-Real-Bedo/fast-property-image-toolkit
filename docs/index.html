How it works
3-step pipeline

From raw upload to pixel-perfect display in under 500ms.

1

Upload

Client sends a multipart POST with the raw image. Multer validates type & size (up to 50 MB).

2

Process

Sharp generates 3 WebP sizes in parallel + a 20px blur placeholder as a Base64 string (~400 bytes).

3

Display

React component shows blur instantly. IntersectionObserver triggers the right-size image when near viewport.

Features
Everything you need

Production-ready, zero compromise.

WebP conversion

30–50% smaller than JPEG at equivalent quality.

3 size variants

Thumbnail 400px · Mobile 800px · Desktop 1920px, all generated in parallel.

Blur placeholder

~400-byte Base64 WebP. No extra HTTP request, instant visual feedback.

Responsive srcset

Browser auto-picks the right resolution. A phone never downloads a 1920px image.

Lazy loading

IntersectionObserver fires 200px before viewport. Off-screen images are never fetched.

EXIF auto-rotate

Sideways phone photos are automatically corrected before processing.

Input validation

MIME type whitelist + configurable 50MB cap. Rejects bad files before reading bytes.

Full TypeScript

Every type exported. Backend JSDoc types sync with frontend interfaces.

Quick start
Up in 3 commands

Clone, configure, run.

Terminal
# 1. Clone and install
git clone https://github.com/The-Real-Bedo/fast-property-image-toolkit.git
cd fast-property-image-toolkit && npm install

# 2. Configure
cp packages/backend/.env.example packages/backend/.env

# 3. Start
npm run dev:backend
# → http://localhost:4000/api/images/process ✅

React usage
import { PropertyImage } from './components/PropertyImage';

function PropertyCard({ property }) {
  return (
    <PropertyImage
      data={property.image}    // JSON from the backend API
      alt={property.title}
      aspectRatio="3/2"
      rootMargin="200px"      // preload 200px before viewport
    />
  );
}
