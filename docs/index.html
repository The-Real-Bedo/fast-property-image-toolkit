What's included
Everything you need

Production-ready, zero compromise.

WebP conversion

30–50% smaller than JPEG at equivalent quality. Sharp handles the heavy lifting.

3 size variants

Thumbnail 400px · Mobile 800px · Desktop 1920px — all processed in parallel.

Blur placeholder

~400 byte Base64 WebP shown instantly — zero extra HTTP request needed.

Responsive srcset

Browser picks the right resolution automatically. Phones never load 1920px images.

Lazy loading

IntersectionObserver fires 200px before viewport. Off-screen images never download.

EXIF auto-rotate

Sideways photos from phones and cameras fixed automatically before processing.

Input validation

MIME type whitelist + 50 MB cap. Bad files rejected before any bytes are read.

Full TypeScript

Every type exported. Frontend interfaces mirror backend JSDoc types exactly.

Get started
Up and running in 3 commands

Clone, configure, run.

Terminal
# 1. Clone and install
git clone https://github.com/The-Real-Bedo/fast-property-image-toolkit.git
cd fast-property-image-toolkit && npm install

# 2. Configure
cp packages/backend/.env.example packages/backend/.env

# 3. Start the server
npm run dev:backend
# → http://localhost:4000/api/images/process ✅
React component
import { PropertyImage } from './components/PropertyImage';

function PropertyCard({ property }) {
  return (
    <PropertyImage
      data={property.image}    // JSON from the backend API
      alt={property.title}
      aspectRatio="3/2"
      rootMargin="200px"       // preload 200px before viewport
      onLoad={() => console.log('Image ready!')}
    />
  );
}
